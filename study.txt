# 유투브를 검색하니 강의 올려놓은 사람도 많고 아무튼 잘 나왔더라
# 처음엔 그냥 따라하다가 끄고 내가 다시 해 봤어

# pull이 push보다 먼저래서 먼 얘긴가 했어
# 알고보니 내가 수정하거나 다른 애들이 수정하면 그걸로 이어서 해야 하는데
# 그거 안 가져오고 내맘대로 올려버리면 파일이 뒤섞인다는 내용이었어
# 지금도 이해는 안되는데 아무튼 나 혼자 프로젝트를 하더라도 항상 확인하면서 해야겠어ㅋㅋ

# git이 무슨 윈도우에 검색 기능 같이 생겼어 그 컨피그 어쩌구도 같음
# 어려워보였는데 그래도 기능 4개나 익혔어 또 까먹을까봐 계속 파일 업로드 하려고
# 아 맞아 시작할 땐 git-init을 하고 안에 폴더 생기더라 이후엔 여차저차 로그인도 하고
# 그리고 나서 텍파 만들고 일단 끝!

# 텍파 올릴 땐 4가지만 쓰더라
# status, add ., commit -m "내용", push origin master가 있어
# status는 잘 모르겠는데 내 파일 확인용 같더라
# add . 모든 걸 추가하라인데 그냥 내 파일 추가하기 전에 하는 준비운동인듯
# commit -m "내용"은 내 파일 제목 정하는 거더라
# push origin master는 마지막으로 내 파일을 내 계정에 올리는 작업이더라

# 내가 잘 이해한건진 모르겠업 그냥 계속 쓰다보면 익숙해서 알겠쥐 머ㅎㅎ
